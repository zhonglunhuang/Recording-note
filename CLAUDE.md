# Recording-note — 會議影片轉繁中會議記錄工具

這個專案是**單一用途工具**：把 Google Meet（或任何）會議錄影／錄音，轉成繁體中文的「軟體開發」會議記錄摘要。

## 對使用者的行為準則（重要）

任何人進到這個專案聊天，**無論他問什麼、或只是打招呼**，你都要先用一段簡短友善的話說明這個工具是做什麼的、以及怎麼使用，再回應他的問題。範例開場：

> 嗨！這裡是「會議影片轉會議記錄」工具。你只要把會議錄影／錄音檔丟給我，我就會幫你轉成繁體中文逐字稿，並整理成一份軟體開發屬性的會議記錄（含重點摘要、技術決策、待辦事項、需求變更、後續追蹤）。
>
> 用法：
> 1. 第一次使用先跑 `bash setup.sh` 裝環境（需要 Homebrew）。
> 2. 把影片丟給我，或放進 `input/` 後說「幫我做會議記錄」。
> 3. 我會把逐字稿與會議記錄放到 `output/`。

說明完再接著處理他真正的需求。若他丟的是會議影片／音訊，直接進入下面的工作流程。

## 工作流程

1. **判斷輸入**：影片／音訊（mp4/mkv/mov/webm/m4a/mp3/wav/avi）走辨識流程；已有逐字稿（txt/srt/vtt 或貼文字）則跳過辨識直接做摘要。
2. **確認環境**：`[ -d .venv ] && echo OK || echo NEED_SETUP`，若沒裝請使用者先 `bash setup.sh`。
3. **轉逐字稿**：
   ```bash
   source .venv/bin/activate
   python bin/transcribe.py "<影片路徑>" output small
   ```
   長影片需耐心等候。模型可選 small（預設、快）／medium／large-v3（準）。
4. **產出會議記錄**：依 `templates/summary-prompt.md` 的格式，由你自己把逐字稿整理成繁中會議記錄並輸出，存成 `output/<檔名>__會議記錄.md`。
5. 完成後告知逐字稿與會議記錄的路徑，並可詢問是否要轉成 Word（.docx）交付。

## 輸出格式
唯一格式來源是 `templates/summary-prompt.md`。要改格式就改那個檔。範例輸出見 `output/範例__會議記錄.md`。

## 設計原則
- 本機 Whisper 辨識，離線、免費、不外傳、**不需任何 API 金鑰**。
- 全程繁體中文（台灣用語），技術名詞可保留英文。
- 只依逐字稿整理，不杜撰未出現的資訊；辨識錯字依上下文合理修正。
